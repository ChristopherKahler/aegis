<objective>
## Goal
Create 8 application and synthesis domain knowledge modules covering the application-layer audit concerns (correctness, security, compliance, testing) and the meta/synthesis concerns (maintainability, change risk, team risk, risk forecasting).

## Purpose
These 8 domains complete the full 14-domain knowledge base that agents use for codebase auditing. Without these domains, AEGIS cannot audit application correctness, security posture, compliance exposure, test strategy quality, code health, change risk, team knowledge distribution, or synthesize cross-domain risk. Combined with Plan 03-01's 6 infrastructure domains, this delivers the complete domain knowledge layer.

## Output
- 8 domain files in `src/domains/`
- Each with 8 sections: Overview, Audit Questions, Failure Patterns, Best Practice Patterns, Red Flags, Tool Affinities, Standards & Frameworks, Metrics
- Strict 1:1 failure-to-best-practice pattern mapping (learned from 03-01 verification)
</objective>

    - Best Practice Patterns: 1:1 matching
