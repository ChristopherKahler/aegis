## Goal
Create 4 extended tool adapter specifications (Checkov, Syft, Grype, git-history) completing Phase 4's full tool integration stack. All must conform to the tool convention and follow the patterns established in Plan 04-01.

## Purpose
Completes AEGIS's 7-tool analysis stack. Checkov adds IaC scanning (Domains 04, 05). Syft+Grype add SBOM generation and dependency vulnerability scanning (Domains 04, 05). Git-history mining adds change frequency, ownership concentration, and churn analysis (Domains 11, 12) — critical for both Core diagnosis and Transform change-risk modeling.

## Output
4 tool adapter specification files in `src/tools/`:
- `checkov.md` — IaC security and compliance scanning
- `syft.md` — SBOM generation (Software Bill of Materials)
- `grype.md` — Vulnerability scanning against SBOM
- `git-history.md` — Git history mining for change risk, ownership, and churn signals

    Create all 4 tool adapter files following the convention in docs/standards/tools.md and the patterns from Plan 04-01.

    **For each tool:**
    - YAML frontmatter with all required fields
    - All 6 body sections in correct order
    - Multiple installation methods (pip/brew/binary + Docker)
    - Docker execution variant
    - Realistic JSON output examples (not placeholder/generic)
    - Complete normalization field mapping table with all 4 dimensions
    - Comprehensive limitations (cannot-detect, false positives, false negatives)

    **Tool-specific guidance:**

    **checkov.md** (type: iac_scan, domains: 04, 05)
    - Bridgecrew/Checkov for IaC scanning (Terraform, CloudFormation, Kubernetes, Dockerfile, etc.)
    - Signal ID pattern: S-CHK-{NNN}
    - Focus: misconfiguration, compliance violations, security posture
    - pip install checkov, brew install checkov, Docker variant

    **syft.md** (type: sbom, domains: 04, 05)
    - Anchore Syft for SBOM generation
    - Signal ID pattern: S-SYF-{NNN}
    - Focus: dependency inventory, license identification — produces the SBOM that Grype consumes
    - Note: Syft signals are inventory (not vulnerabilities) — normalization is different from scanner tools
    - curl/brew install, Docker variant

    **grype.md** (type: vulnerability_scan, domains: 04, 05)
    - Anchore Grype for vulnerability scanning against SBOMs
    - Signal ID pattern: S-GRP-{NNN}
    - Focus: known CVE matching against dependency inventory
    - Can scan directly OR consume Syft SBOM output — document both modes
    - curl/brew install, Docker variant

    **git-history.md** (type: history_mining, domains: 11, 12)
    - Git-native commands (git log, git shortlog, git diff --stat) — no external tool install needed
    - Signal ID pattern: S-GH-{NNN}
    - Focus: file churn frequency, author concentration, commit patterns, code age, hotspot detection
    - install_required: false (uses git, which is assumed present)
    - MUST include change-risk normalization mapping per Transform tool conventions
    - This tool feeds both Core (Domains 11, 12) and Transform (Change Risk Modeler)

    **Avoid:**
    - Embedding interpretation/audit logic (tools describe output, not meaning)
    - Generic placeholder output examples — use realistic tool-specific JSON
    - Single-OS-only install commands
    - Omitting Docker variants
  </action>
  <verify>Read each file and verify: frontmatter complete, all 6 sections present, installation options include multi-platform + Docker, JSON output examples are realistic, normalization table maps all 4 dimensions, limitations section has cannot-detect + false positives + false negatives</verify>

</task>
