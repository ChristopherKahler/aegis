  <action>
    Create the session handoff workflow following docs/standards/workflows.md convention exactly.

    **Purpose:** Define how context passes between agent sessions across AEGIS audit phases. Each agent runs in its own Claude Code session. Findings, disagreements, and state must persist between sessions.

    **Required sections (all XML tags):**
    - purpose: One-sentence description
    - phase_context: This is a utility workflow, not phase-specific. Describe its role across all phases.
    - required_input: @-references to .aegis/ state files, schema files, manifest
    - process: Steps for (1) saving session output, (2) validating against schemas, (3) writing to .aegis/ artifact directory, (4) updating .aegis/STATE.md, (5) preparing next session's input context
    - output: What artifacts are created/updated
    - error_handling: Session timeout, partial output, schema validation failure, corrupt state

    **Key concerns the workflow must address:**
    - What state persists between sessions (findings, disagreements, signals, state file)
    - What context is loaded into a new session (manifest, persona, domains, prior findings)
    - What is lost at session boundary (in-memory reasoning, partial analysis)
    - How session output is validated before being committed to .aegis/
    - How .aegis/STATE.md is updated after each session
    - How to resume an interrupted session

    **Step structure:**
    ```xml
    <step name="step_name" priority="first|blocking|optional">
    Numbered instructions
    </step>
    ```

    **Avoid:** Embedding domain knowledge or persona behavior. The workflow orchestrates; it does not analyze.
