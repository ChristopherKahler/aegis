<objective>
## Goal
Create the 4 Transform slash commands that serve as user entry points for the AEGIS remediation and evolution pipeline: `/aegis:transform`, `/aegis:remediate`, `/aegis:playbook`, and `/aegis:guardrails`. Each command enforces safety prerequisites (complete Layer A, intervention level display, user confirmation) before delegating to Transform workflows.

## Purpose
Transform commands bridge the gap between diagnostic findings (Layer A) and actionable remediation (Layers B & C). Without them, the Transform pipeline has no user-facing entry point. These commands enforce the safety framework — no Transform output is produced without the user seeing intervention levels and confirming.

## Output
4 command files in `src/transform/commands/` following the command convention (YAML frontmatter + 5 XML sections) with Transform-specific safety requirements.
</objective>

  <action>
    Create `src/transform/commands/` directory and all 4 command files.

    **All commands share this structure:**
    - YAML frontmatter: `name`, `description`, `argument-hint` (optional)
    - 5 XML sections: `<objective>`, `<execution_context>`, `<context>`, `<process>`, `<success_criteria>`

    **All Transform commands share these safety requirements:**
    - Step 1 MUST check .aegis/report/ exists (Layer A complete) — refuse to proceed if incomplete
    - MUST display intervention level distribution before producing any output
    - MUST require explicit user confirmation before Transform produces output at Planning level or above
    - Reference transform-safety workflow for validation

    **transform.md** (`/aegis:transform`):
    - argument-hint: none
    - The master Transform command — initiates the full Transform pipeline (Phases 6-8)
    - Process flow:
      1. Check prerequisites: verify .aegis/report/ exists (all Core phases 0-5 complete)
      2. Display Transform scope: finding count, estimated playbook count, confidence distribution, maximum intervention level available
      3. Select Transform scope: Full (all findings, Phases 6-8) / Selective (choose domains/findings) / Playbooks only (Phase 6 only) / Cancel
      4. Display safety info: intervention level distribution (Suggesting: N, Planning: M, Authorizing: K), confidence summary
      5. Require explicit confirmation: "Transform will produce remediation at these intervention levels. Confirm to proceed."
      6. Delegate to phase-6-remediation workflow to begin
    - execution_context references: all 4 Transform workflows
    - context references: .aegis/STATE.md, .aegis/report/, .aegis/findings/

    **remediate.md** (`/aegis:remediate`):
    - argument-hint: `"[domain-number]"`
    - Generates full remediation plan for all findings or a specific domain
    - Process flow:
      1. Check prerequisites: Layer A complete
      2. If $ARGUMENTS has domain number: scope to that domain's findings
      3. If no arguments: scope to all findings
      4. Check existing remediation: if .aegis/remediation/ exists, offer view/regenerate
      5. Display safety info: intervention levels, confidence distribution for scoped findings
      6. Confirm and delegate to phase-6-remediation workflow
      7. After Phase 6, optionally continue to Phase 7 (risk validation) and Phase 8 (execution planning)
    - execution_context references: phase-6-remediation, phase-7-risk-validation, phase-8-execution-planning, transform-safety
    - context references: $ARGUMENTS, .aegis/STATE.md, .aegis/report/, .aegis/findings/, .aegis/remediation/

    **playbook.md** (`/aegis:playbook`):
    - argument-hint: `"<finding-id>"`
    - Generates a remediation playbook for a single specific finding
    - Process flow:
      1. Check prerequisites: Layer A complete
      2. Require $ARGUMENTS with finding-id (error if missing)
      3. Validate finding-id exists in .aegis/findings/
      4. Check existing playbook: if .aegis/remediation/playbooks/{finding-id}.md exists, offer view/regenerate
      5. Display safety info: finding confidence, intervention level for this finding
      6. Confirm and delegate to phase-6-remediation workflow (scoped to single finding)
      7. Display playbook location when complete
    - execution_context references: phase-6-remediation, transform-safety
    - context references: $ARGUMENTS, .aegis/STATE.md, .aegis/findings/{finding-id}, .aegis/remediation/playbooks/

    **guardrails.md** (`/aegis:guardrails`):
    - argument-hint: `"[format]"`
    - Generates project rules from audit findings (for CLAUDE.md, .cursorrules, etc.)
    - Process flow:
      1. Check prerequisites: Layer A complete, AND Layer B remediation exists (.aegis/remediation/ must exist — guardrails derive from remediation knowledge)
      2. If remediation incomplete: route to /aegis:remediate first
      3. Select guardrail format: Claude MD rules / Cursor rules / Both / Cancel
      4. Display safety info: patterns being codified, intervention levels
      5. Confirm and delegate to phase-7-risk-validation workflow (guardrail-generator portion)
      6. Display guardrail file locations when complete
    - execution_context references: phase-7-risk-validation, transform-safety
    - context references: $ARGUMENTS, .aegis/STATE.md, .aegis/remediation/, .aegis/remediation/guardrails/

    **Avoid and WHY:**
    - Do NOT embed agent logic — commands delegate to workflows
    - Do NOT skip safety prerequisites — Transform without Layer A is dangerous (remediation from partial information)
    - Do NOT produce output without intervention level display — violates safety framework
    - Do NOT skip cancellation options — users must never feel trapped
    - Do NOT allow auto-execution — Transform produces plans only, PAUL executes
