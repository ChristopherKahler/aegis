<objective>
## Goal
Convert all 8 AEGIS command specs and 12 workflow files from repo-local `@src/` references to installable `@~/.claude/aegis/` references. Restructure commands into a separate top-level `commands/` directory matching the install target layout. Add `allowed-tools` frontmatter to all commands following PAUL conventions.

## Purpose
This is the foundational step of v0.2 — nothing can be installed until the files reference the correct install-time paths. Commands must be separated from framework files since they install to different locations (`~/.claude/commands/aegis/` vs `~/.claude/aegis/`).

## Output
- 8 command files in `commands/` (new top-level directory) with:
  - `allowed-tools` frontmatter
  - All `@src/` refs rewritten to `@~/.claude/aegis/`
- 12 workflow files in `src/` with all `@src/` refs rewritten to `@~/.claude/aegis/`
- `src/core/commands/` and `src/transform/commands/` removed (commands live in `commands/` now)
- Total: 20 files modified, 8 files moved
</objective>
