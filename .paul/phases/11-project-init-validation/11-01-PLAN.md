<objective>
## Goal
Create two new AEGIS commands — `/aegis:init` (project setup) and `/aegis:validate` (tool testing) — and update `/aegis:audit` to delegate initialization to `/aegis:init` instead of doing it inline.

## Purpose
Currently `/aegis:audit` creates `.aegis/` inline during its wizard flow. This couples project setup with audit execution. Separating init enables: (1) users to set up a project without immediately running an audit, (2) the validate command to run against an initialized project, and (3) cleaner command responsibilities. The validate command gives users confidence that their tool installation actually works before committing to a multi-session audit.

## Output
- `commands/init.md` — `/aegis:init` slash command
- `commands/validate.md` — `/aegis:validate` slash command
- `commands/audit.md` — Updated to delegate init to `/aegis:init`
</objective>

```gherkin
Given the user runs /aegis:init in a target repository

```

    Create a new slash command following the pattern in commands/audit.md and commands/status.md.

    **Frontmatter:**
    - name: aegis:init
    - description: Initialize AEGIS in a target project
    - argument-hint: "[path-to-repo]"
    - allowed-tools: [Read, Write, Edit, Bash, Glob, Grep, AskUserQuestion]

    **Process:**

    1. **Determine target repository:**
       - If $ARGUMENTS has a path, use it
       - Otherwise use current working directory
       - Verify it's a git repository (check .git/ exists)
       - If not a git repo, warn and ask to confirm

    2. **Check for existing .aegis/:**
       - If .aegis/STATE.md exists:
         - Read STATE.md to determine status
         - Display current state summary
         - Offer: [1] Resume existing (run /aegis:resume), [2] Fresh start (archive .aegis/ to .aegis-backup-{timestamp}/), [3] Cancel
         - Wait for user decision
       - If no .aegis/: proceed to create

    3. **Create .aegis/ directory structure:**
       ```
       .aegis/
       ├── STATE.md         # Audit state tracking
       ├── MANIFEST.md      # Version-locked framework references
       └── findings/        # Output directory for phase findings
       ```

    4. **Initialize STATE.md with:**
       ```markdown
       # AEGIS Audit State

       ## Target
       Repository: {repo-name}
       Path: {absolute-path}
       Initialized: {ISO timestamp}

       ## Status
       Overall: initialized
       Current Phase: none (run /aegis:audit to begin)

       ## Phases
       | Phase | Name | Status |
       |-------|------|--------|
       | 0 | Context & Threat Modeling | pending |
       | 1 | Automated Signal Gathering | pending |
       | 2 | Deep Domain Audits | pending |
       | 3 | Change Risk & Reality Gap | pending |
       | 4 | Adversarial Review | pending |
       | 5 | Synthesis & Report | pending |

       ## Resume Info
       Last action: Project initialized
       Next action: Run /aegis:audit to begin diagnostic audit
       ```

    5. **Initialize MANIFEST.md with:**
       - AEGIS version (read from install or framework metadata)
       - Framework path (~/.claude/aegis/)
       - Commands path (~/.claude/commands/aegis/)
       - List of installed tools (detect which are available via command -v / docker image inspect)
       - Timestamp

    6. **Create findings/ directory** (empty, for phase outputs)

    7. **Add .aegis/ to .gitignore if not already present:**
       - Check if .gitignore exists and contains .aegis/
       - If not, append `.aegis/` line
       - Report what was done

    8. **Display summary:**
       ```
       ════════════════════════════════════════
       AEGIS Project Initialized
       ════════════════════════════════════════

       Target: {repo-name} ({path})
       State:  .aegis/STATE.md
       Tools:  X/7 detected

             Run /aegis:validate to verify tool setup

       ```

       | Semgrep | `semgrep --version` | Check venv: `~/.local/share/aegis/venvs/semgrep/bin/semgrep --version` |

       | Syft | `syft version` | - |
       | Grype | `grype version` | - |
       | Git (built-in) | `git --version` | Always available |

       For Python tools installed via venv, check both PATH and venv location.
       For SonarQube, check all three: native CLI, Docker scanner image, Docker server.

    3. **Report results:**
       ```
       ════════════════════════════════════════
       AEGIS Validation Report
       ════════════════════════════════════════

       Framework:
         ✓ ~/.claude/aegis/ (N files)

         ✓ semgrep 1.x.x (venv)

         ✓ checkov 3.x.x (venv)
         ✓ syft 1.x.x
         ✓ grype 0.x.x
         ✓ git 2.x.x (built-in)
       ```

       - Binary tools: "Check ~/.local/bin/ is on PATH — re-run install.sh to reinstall"
