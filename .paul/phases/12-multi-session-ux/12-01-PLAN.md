<objective>
## Goal
Create a smooth multi-session audit experience with automatic checkpoints between phases, enhanced resume flow, and clear progress visibility — so users can walk away after any phase and pick up exactly where they left off.

## Purpose
AEGIS audits span 6+ Core phases and 3+ Transform phases, each potentially requiring its own Claude Code session. Without explicit checkpoint UX, users must manually track where they are. This plan adds the "session management layer" that makes multi-session audits feel seamless.

## Output
- New: `src/core/workflows/phase-checkpoint.md` — checkpoint workflow invoked between phases
- Modified: `commands/audit.md` — multi-phase orchestration loop with checkpoints
- Modified: `commands/resume.md` — smarter context restoration with session diff
- Modified: `commands/status.md` — estimated remaining sessions + phase timing
- Modified: `commands/init.md` — STATE.md template with session tracking fields
</objective>


  <name>Task 1: Create phase-checkpoint workflow and update STATE.md template</name>
  <files>src/core/workflows/phase-checkpoint.md, commands/init.md</files>
  <action>
    **Create `src/core/workflows/phase-checkpoint.md`:**

    A workflow invoked after each phase completes, before the next phase begins. Structure:

    1. **Capture Phase Completion** (first priority)
       - Read `.aegis/STATE.md` to identify which phase just completed
       - Count findings produced in this phase (from `.aegis/findings/`)
       - Count disagreements detected (from `.aegis/disagreements/`)
       - Record phase completion timestamp in STATE.md
       - Increment session_count in STATE.md

    2. **Display Phase Summary** (blocking)
       - Show completed phase name + duration
       - Show finding count (by severity: critical/high/medium/low)
       - Show disagreement count (if any)
       - Show cumulative progress: "Phase N of M complete"
       - Show progress bar: `[████████░░] 80%`

    3. **Preview Next Phase** (blocking)
       - Name and description of next phase
       - Number of agents that will run
       - Estimated sessions needed (1 for phases 0,1,4,5; up to 8 for phase 2; 2 for phase 3)
       - Prerequisites satisfied? (should always be yes at this point)

    4. **Offer Checkpoint Options** (blocking)
       - Present exactly 3 options:
         ```
         [1] Continue to Phase N+1
         [2] Pause here (safe to close session)
         [3] Abort audit (preserves all work so far)
         ```
       - On "continue": invoke next phase workflow
       - On "pause": update STATE.md with `overall: paused`, `last_action`, `next_action`, display "Run /aegis:resume when ready"
       - On "abort": update STATE.md with `overall: paused`, display findings count and how to resume later

    5. **Update STATE.md** (required)
       - Add to checkpoint_history: `{phase, timestamp, action_taken}`
       - Update phase_timestamps with completion time
       - If paused: set clear resume instructions

    **Conventions:**
    - Follow same markdown workflow format as existing `src/core/workflows/session-handoff.md`
    - Use XML-like step tags with priority attributes
    - Include error handling for missing STATE.md fields

    **Update `commands/init.md`:**
    - Add session tracking fields to the STATE.md template in Step 4:
      ```markdown
      ## Session Tracking
      - Sessions: 0
      - Last session: -
      - Started: -

      ## Checkpoint History
      (Populated as phases complete)
      ```
    - Keep all existing init.md content — only add the new section to the STATE.md template
    - The new section goes after "## Resume Info" and before the end of the template

    **Do NOT:**
    - Modify any phase orchestration workflow files (phase-0 through phase-5)
    - Change the session-handoff workflow
    - Add fields that duplicate what STATE.md already tracks
  </action>
